# Techsprint Rules and Regulations

## General Rules

### Eligibility
1. All participants must be currently enrolled students of the college
2. Each team must have 2-4 members
3. Students can only be part of one team

### Registration
1. Teams must register before the registration deadline
2. Team composition cannot be changed after registration closes
3. All team members must be present during the event

### Project Requirements
1. Projects must be original work created during the hackathon
2. Teams can use existing libraries, frameworks, and APIs
3. Pre-existing code or projects are not allowed
4. All code must be written during the hackathon period

### Submission Guidelines
1. Projects must be submitted before the deadline
2. Late submissions will not be accepted
3. All submissions must include:
   - Complete source code
   - README with documentation
   - Installation and usage instructions
   - List of technologies used

### Conduct
1. All participants must follow the Code of Conduct
2. Respect other teams and their work
3. Plagiarism will result in immediate disqualification
4. Cheating in any form is prohibited

## Technical Rules

### Development
1. Use any programming language or framework
2. Projects must be functional (working demo required)
3. Open-source libraries and tools are allowed
4. Proper attribution must be given for third-party resources

### Submission Format
1. Submit code via GitHub repository
2. Include a detailed README.md file
3. Provide setup and running instructions
4. Include screenshots or demo video (recommended)

### Intellectual Property
1. Participants retain rights to their code
2. Organizers may showcase projects for promotional purposes
3. Teams should use open-source licenses (e.g., MIT, Apache 2.0)

## Judging

### Evaluation Criteria
Projects will be judged on:
1. **Innovation** (30%) - Creativity and uniqueness
2. **Technical Implementation** (25%) - Code quality and complexity
3. **Functionality** (20%) - Working features
4. **Impact** (15%) - Real-world applicability
5. **Presentation** (10%) - Demo and documentation quality

### Presentation
1. Each team gets 5-7 minutes to present
2. Followed by 2-3 minutes of Q&A
3. All team members should be present
4. Judges' decisions are final

## Prizes

### Awards
1. Cash prizes for top 3 positions
2. Special category awards
3. Certificates for all participants
4. Goodies from sponsors

### Disqualification
Teams may be disqualified for:
- Plagiarism or copying code
- Violating the Code of Conduct
- Missing presentation without prior notice
- Submitting incomplete or non-functional projects
- Not following submission guidelines

## Important Notes

1. **Internet Access**: WiFi will be provided during the event
2. **Power Supply**: Bring your chargers and adapters
3. **Food**: Meals and refreshments will be provided
4. **Support**: Mentors will be available for guidance
5. **Emergency**: Contact organizers for any urgent issues
